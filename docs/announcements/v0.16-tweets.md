## LinkedIn Post (Longer Form)

> **Stentorosaur v0.16: Finally, Private Repo Support for Docusaurus Status Pages**
>
> I'm excited to announce Stentorosaur v0.16, which solves a long-standing limitation: private repository support for runtime status updates.
>
> **The Problem**
> Docusaurus status pages using raw.githubusercontent.com URLs would silently fail for private repositories, forcing users into build-time-only updates.
>
> **The Solution**
> The new `dataSource` configuration offers four flexible strategies:
> 1. **GitHub** - Direct fetch for public repos
> 2. **HTTP** - Universal adapter for any JSON endpoint (including your own proxy)
> 3. **Static** - Local files for monorepo setups
> 4. **Build-only** - No runtime fetch
>
> For private repos, we recommend a lightweight Cloudflare Worker or Vercel Edge Function that adds authentication server-side.
>
> **Security First**
> We've added warnings for common mistakes: HTTP URLs on HTTPS sites, exposed auth headers, and oversized payloads.
>
> **Migration**
> Fully backward compatible. Your existing `fetchUrl` config continues to work.
>
> Install: `npm i @amiable-dev/docusaurus-plugin-stentorosaur@0.16.0`
>
> #opensource #docusaurus #typescript #webdev #devtools
